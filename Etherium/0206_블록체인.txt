[블록체인]
블록체인(Block Chain) : 분산 컴퓨팅 기술 기반의 데이터 위변조 방지 기술
한국에서는 블록체인 기반 암호화폐의 줄임말로 많이씀

P2P(Peer To Peer의 줄임말로 중앙 서버를 거치지않고 클라이언트 컴퓨터끼리 직접 통신하는 방식)
	방식을 기반으로 해서 소규모 데이터들이 사슬(체인) 형태로 무수히 연결되어 형성된 '블록'이라는
	 분산 데이터 저장 환경에 대상 데이터를 저장함으로써 누구도 임의로 수정할 수 없이 누구나 변경의
	  결과를 열람할 수 있게끔 하는 기술

이 P2P형태로 작동하는 애플리케이션을 디앱(dAPP, Decentralized Application) 이라고함
	= 탈중앙화 애플리케이션이라고도 하는데, 블록체인에서 중요한 개념이 탈중앙화

[탈중앙화의 장점]
	1.익명성
		모든 거래는 기록되지만, 그 데이터를 수집하거나 직접 확인할 수는 없어서
		 지갑과  화폐의 주인이 누구인지 특정할 수 없음
	2. 보안성
		시스템 보안이 훌륭한 편
		중앙화된 시스템은 정보를 특정한 장소에 모아두기 때문에 집중적으로 공략할 곳을 
		 특정하기 쉬운반면, 블록체인 시스템은 수많은 독립적인 요소로 이루어져 있어
		 시스템 장애가 발생해도 자료 손실의 위험이 적고, 보안의 급소를 특정하기 어려움

[탈중앙화의 단점]
	1. 낮은 유동성
		이 시스템의 비전은 밝지만, 대중들은 여전히 기존 금융 시스템에 익숙하기 때문에
		 = 낮은인식
	2.책임이 불분명함
		화폐가 증발하거나 도난당한 경우, 그 사건을 책임질 사람 / 기관을 특정하기 어려움

이 dAPP은 새로운 기술이기에 앞으로 성공할지 예측하기는 어렵지만,  다양한 분야에서 시도해보는 중인데
	원산지 / 원본 추적기능
	신원확인
	소유권 증명
	크라우드 펀딩
등이 있음

[용어 설명]
	-주소(address)	 	: 공개키 암호로 암호화폐를 주고 받는 문자열
	-지갑(wallet) 	 	: 거래에 필요한 개인키를 저장하는 공강
	-거래(transaction)   	: 코인과 토큰 소유권을 포함하고 있는 데이터가 기록되고, 작성자의 전자 서명을
					통해 코인 / 토큰의 발행을 증명 
	-컨펌(confirm)		: 거래를 블록에 저장시켜서 기존의 거래내역과 새로운 거래내역을 검증하는 것
					컨펌의 횟수가 많을수록 확정한 거래내역의 신뢰성이 보장됨
	-코인(coin)/토큰(token) 	: 메인넷에서 발행한 암호화폐를 코인
				  블록체인 시스템을 통해 발행한 것을 토큰 
	-메인넷(mainnet) / 테스트넷(testnet) : 공식적인 프로그램 출시 및 운용하는 메인 네트워크를 메인넷,
					개발시에 사용하는 테스트넷

=============================================================================

개발을 위해서 솔리디티 라는 언어를 사용

솔리디티(Solidity) : 
	이더리움에서 제공하는 스마트 컨트랙트 개발언어
	스마트 컨트랙트 : 계약 당사자 간 거래 내용을 코드로 기록해서 블록체인에 올리면,
			계약 조건이 충족됏을때 계약을 자동으로 이행해주는 시스템
			(=블록체인에서 작동하는 디지털화 된 계약서)

기본적으로 이더(ETH)를 보내고 받는데 필요한 데이터 타입, 함수 등을 제공
특징은 크게 3가지
	1. 정적 타입 언어 : 컴파일시 변수의 타입이 결정되는 언어
			개발자가 변수에 들어갈 값의 형태에 따라 직접 타입을 명시해주어야 함

	2. Etherium Virtual Machine(EVM)에서 돌아감
		EVM이란 이더리움 블록체인 네트워크의 참여자가 공유하는 하나의 가상머신이며
			이더리움을 작동하는 엔진

	3. 튜링 완정성 : 튜링완전이란, 어떤 프로그래밍 언어나 추상 머신들이 튜링 머신과 동일한 계산능력으로
			문제를 풀고 해결할 수 있다는 의미
		         솔리디티는 자체적인 튜링 완전 언어들을 지원하기 때문에 사실상 가능한 모든 형태의
			거래를 코드로 작성할 수 있음
		        또, 컴퓨터 코드 작업마다 수수료인 가스(Gas)를 부과해서 악의적인 공격에 방어할 수 있음
